## 🔌 Integration Methods (Choose One)

### Option 1: REST API (Polling)
```javascript
// Simplest approach
setInterval(async () => {
  const res = await fetch(`/api/courier/${id}`);
  const data = await res.json();
  tracker.updateCourierPosition(data.lat, data.lon, data.heading, data.speed);
}, 2000);
```
✅ Simple | ✅ Works everywhere | ⚠️ Slight delay

### Option 2: WebSocket (Real-time)
```javascript
// Best performance
const ws = new WebSocketGPSStream(tracker, orderId, {
  wsUrl: 'wss://api.turon.uz/gps',
  authToken: token
});
ws.connect();
```
✅ Real-time | ✅ Low latency | ✅ Always connected

### Option 3: Firebase
```javascript
// Google Firebase
const listener = new FirebaseGPSListener(tracker, orderId, {
  database: firebaseDB
});
listener.start();
```
✅ Easy setup | ✅ Google-backed | ✅ Scalable

### Option 4: Supabase
```javascript
// Open-source Firebase alternative
const listener = new SupabaseGPSListener(tracker, orderId, {
  supabase: supabaseClient
});
listener.start();
```
✅ PostgreSQL | ✅ Open source | ✅ Full control

### Option 5: Auto-Select Best (Hybrid)
```javascript
// Automatically tries each in order
const gps = new HybridGPSManager(tracker, orderId, {
  wsUrl: 'wss://api.turon.uz/gps',
  apiBase: 'https://api.turon.uz',
  authToken: token
});
gps.start();
```
✅ Reliable | ✅ Automatic fallback | ✅ Best UX

## 🎯 Core Method

```javascript
// THE main function you'll use
tracker.updateCourierPosition(latitude, longitude, azimuth, speed);

// Example: Courier heading northeast at 20 km/h
tracker.updateCourierPosition(41.32, 69.25, 45, 20);
// 0° = North, 90° = East, 180° = South, 270° = West

// Example: Heading north (straight up)
tracker.updateCourierPosition(41.32, 69.25, 0, 15);

// Example: Heading south (straight down)
tracker.updateCourierPosition(41.32, 69.25, 180, 15);
```

## 💡 Pro Tips

1. **Always test simulation first** before real GPS
2. **WebSocket > polling** for better UX
3. **Update every 2-3 seconds** (not faster)
4. **Use actual heading** from GPS device
5. **Monitor on mobile early** to catch issues
6. **REST API as fallback** from WebSocket
7. **Validate all coordinates** before update
8. **Error handling** prevents bad UX

## 🚨 Common Issues & Solutions

| Issue | Solution |
|-------|----------|
| Blank white map | Check API key in script |
| Icon not rotating | Verify azimuth 0-360° |
| Route not calculating | Ensure coordinates valid |
| Nothing updating | Call updateCourierPosition() |
| Mobile cuts off | Refresh page, check viewport |
| Performance lag | Reduce update frequency |
| WebSocket fails | Falls back to REST auto |
| Dark mode broken | Check prefers-color-scheme |
